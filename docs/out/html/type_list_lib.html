Type List Library

Deals with the transformation and querying of type-only class template parameter-lists.

Nomenclature

Throughout this document there will be certain used terminology that is not defined in the C++ standard, this section is to shed some light on any upcoming potential confusion.

Type List

Any mention of the term type list in this document refers to the parameter-list of an instantiated class template that consists solely of types, any non-type or template type template argument is excluded from this wording and thus, if part of a template parameter-list, shall not be defined as part of a type list any longer.

Any mention of type list is solely referring to a parameter-list of an instantiated class template, by that means, the declaration of template parameters in a class template can not be regarded as a type list.

A type T is said to be type list conform, if T is a class template with at least one to (hypothetically) infinite type parameters.

Query Trait

A query trait is a class template that always provides a static value constant that evaluates to a constant expression determining specific attributes of the input given as template arguments to the trait, this guarantee must only be given as long as nothing statically asserted inside such a trait.

Transformative Trait

A transformative trait is a class template that always provides a member alias type which aliases the transformed type based on the input given as template arguments to the trait, this guarantee must only be given as long as nothing statically asserted inside such a trait.

Summary

The Type List Library deals with the transformation and identification of type lists[1].

A good example of a type list conform[1] type is std::tuple, which is a class template that can take an arbitrary amount of type template arguments.

ExTrait also provides extrait::T which is a little more lightweight than std::tuple and thus is a perfect fit for any type list[1] related needs, which can be used to inspect its own type list and give back a transformed version.

Another example would be std::vector, std::list or std::basic_string, however, these might have internal language-specific constraints on their template arguments and make them a little difficult to handle, nevertheless, if handled carefully they might be a good fit as well.

Type Traits

The Type-List library comes with a set of easy-to-use and elaborate utility traits that allow transformation[3] and queries[2] to inputted type lists.[1]
